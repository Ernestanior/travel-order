# 旅行社订单管理系统 - 使用说明

## 系统概述

这是一个基于 Next.js 开发的旅行社订单管理系统，用于管理机票预订、换票、客户和供应商信息。系统界面参考了你提供的旧系统截图，采用现代化的网页设计。

## 主要功能模块

### 1. 主菜单（Main Menu）
- 显示系统统计数据：订单数量、收入、未付款金额
- 快速进入各个功能模块

### 2. 预订订单（Booking Order）
**订单列表页面**：
- 显示所有预订订单
- 字段包括：Booking#、Customer、Depart Date、Arrival Date、Total Cost、Paid、Outstanding、Status
- 点击订单可查看详情

**订单详情页面**：
- **客户信息区**：客户名称、地址、电话、传真
- **航班信息区**：
  - Departure（出发）
  - Depart2（第二段出发）
  - Arrival（到达）
  - Arrival 2（第二段到达）
  - 每段包含：日期、时间、航班号、目的地
- **乘客数据**：乘客姓名列表
- **财务信息**：
  - Discount（折扣）
  - Booking Total Amount（总金额）
  - Receipt 表格（收据记录）
- **特殊说明**：Special Instruction

### 3. 换票订单（Exchange Order）
**订单列表页面**：
- 显示所有换票订单
- 字段包括：Exchange#、Agent、Dept Date、Arv Date、Total Cost、Paid、Outstanding、Status、Booking#

**订单详情页面**：
- **供应商信息**：供应商名称、地址、电话、传真
- **预订数据**：关联的预订信息、航班信息
- **乘客数据**：乘客姓名和出生日期
- **Item Data**：换票项目明细（描述、数量、单价、总价）
- **收据记录**：支付记录

### 4. 客户管理（Customer）
- 查看所有客户列表
- 客户信息包括：姓名、电话、地址、邮箱
- 支持搜索功能

### 5. 供应商管理（Supplier）
- 查看所有供应商（航空公司）列表
- 供应商信息包括：名称、电话、传真、地址
- 支持搜索功能

### 6. 报表/打印（Reports / Printouts）
提供多种报表选项：
- A: Booking Inquiry (Departure) - 预订查询（按出发日期）
- B: Exchange Inquiry (Supplier) - 换票查询（按供应商）
- D: Outstanding Transactions Report: Before Departure - 未付款报表（出发前）
- E: Outstanding Transactions Report: By Passenger - 未付款报表（按乘客）
- F: Passenger Inquiry - 乘客查询
- G: Receipt Inquiry - 收据查询
- H: Receipt Transactions Report - 收据交易报表
- I: Printing of Receipt - 打印收据
- J: Printing of Booking - 打印预订单
- K: Printing of EO - 打印换票单
- L: Profit Report - 利润报表
- M: By Customer - 按客户查询
- N: Closed Booking Transaction - 已关闭预订交易

## 快速开始

### 本地运行

1. 安装依赖：
```bash
cd /Users/ern/Desktop/code/airline-order
npm install
```

2. 启动开发服务器：
```bash
npm run dev
```

3. 打开浏览器访问：http://localhost:3000

### 部署到 Vercel

**方法一：使用命令行**
```bash
npm install -g vercel
vercel login
vercel
```

**方法二：通过 GitHub**
1. 把代码推送到 GitHub
2. 在 vercel.com 登录
3. 导入你的 GitHub 项目
4. 点击部署

详细部署步骤请查看 `DEPLOY.md` 文件。

## 数据说明

**当前状态**：系统使用模拟数据（存储在 `lib/mockData.ts` 文件中）

**模拟数据包括**：
- 4个预订订单
- 2个换票订单
- 4个客户
- 3个供应商

**生产环境**：需要连接真实数据库（如 PostgreSQL 或 MongoDB），并创建 API 接口处理数据操作。

## 后续开发建议

如果要投入生产使用，建议添加以下功能：

### 必要功能
1. **数据库集成**：连接 PostgreSQL 或 MongoDB 数据库
2. **用户认证**：添加登录系统，管理员权限控制
3. **API 接口**：创建 RESTful API 处理所有数据操作
4. **表单验证**：确保输入数据的正确性
5. **数据持久化**：所有增删改操作保存到数据库

### 增强功能
1. **PDF 生成**：实现发票和收据的 PDF 导出
2. **邮件通知**：订单确认、支付提醒等邮件
3. **高级搜索**：按多个条件筛选订单
4. **数据导出**：Excel 格式导出报表
5. **打印优化**：优化打印样式
6. **移动端优化**：改进手机浏览体验
7. **多语言支持**：中英文切换

## 常见问题

**Q: 数据会保存吗？**
A: 当前版本使用模拟数据，刷新页面后修改会丢失。需要连接数据库才能持久化保存。

**Q: 如何修改模拟数据？**
A: 编辑 `lib/mockData.ts` 文件。

**Q: 可以打印发票吗？**
A: 当前版本有打印按钮，但需要进一步开发 PDF 生成功能。

**Q: 支持多个用户同时使用吗？**
A: 需要添加数据库和用户认证系统。

**Q: 部署费用是多少？**
A: Vercel 免费计划足够个人和小型团队使用。商业项目建议使用 Pro 计划（$20/月）。
