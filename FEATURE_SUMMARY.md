## ✨ 核心功能

## 🔍 搜索功能详解

### 预订订单搜索

#### 1. 按出发日期搜索
```
输入: 2006-01-15
结果: 显示该日期出发的所有订单
```

#### 2. 未付款订单查询
```
输入: 2006-12-31
结果: 显示该日期前出发且还有未付款的订单
逻辑: departureDate < 指定日期 AND outstanding > 0
```

#### 3. 按客户名称搜索
```
输入: "John" 或 "李明"
结果: 显示所有包含该关键词的客户订单
支持: 模糊匹配，不区分大小写
提示: 自动显示匹配的客户名下拉列表
```

### 交换订单搜索
```
按供应商: 输入供应商名称
支持: 模糊匹配
提示: 自动完成下拉列表
```

### 客户搜索
```
实时搜索: 输入任意文字即时过滤
匹配范围: 客户名称
响应: 即时更新列表
```

### 乘客查询
```
按客户: 输入客户名称
显示: 该客户所有订单及乘客
支持: 模糊匹配，自动提示
```

## 🌐 API端点

### 1. 预订订单API
```bash
# 获取所有订单
GET /api/booking-orders?searchType=all

# 按日期搜索
GET /api/booking-orders?searchType=date&departureDate=2006-01-15

# 未付款订单
GET /api/booking-orders?searchType=outstanding&outstandingBeforeDate=2006-12-31

# 按客户搜索
GET /api/booking-orders?searchType=customer&customer=John
```

### 2. 交换订单API
```bash
# 获取所有交换订单
GET /api/exchange-orders

# 按供应商搜索
GET /api/exchange-orders?supplier=Singapore+Airlines
```

### 3. 客户API
```bash
# 获取所有客户
GET /api/customers

# 搜索客户
GET /api/customers?search=John
```

### 4. 供应商API
```bash
# 获取所有供应商
GET /api/suppliers

# 搜索供应商
GET /api/suppliers?search=Singapore
```

### 5. 统计API
```bash
# 获取业务统计
GET /api/stats

返回示例:
{
  "totalBookings": 28468,
  "totalExchanges": 28518,
  "totalRevenue": 68218605.01,
  "outstandingAmount": -8405698.27
}
```

## 🎯 使用场景

### 日常操作

#### 场景1: 查询某个客户的订单
```
1. 访问 /booking-orders
2. 选择"By Customer"搜索模式
3. 输入客户名称（如"CHAN"）
4. 系统显示所有匹配的订单
5. 点击订单查看详情
```

#### 场景2: 查找未付款订单
```
1. 访问 /booking-orders
2. 选择"Outstanding (Before Date)"
3. 选择截止日期
4. 查看所有该日期前未结清的订单
5. 导出报表或联系客户
```

#### 场景3: 查询乘客信息
```
1. 访问 /passenger-inquiry
2. 输入客户名称
3. 查看该客户所有订单的乘客列表
4. 点击订单号查看详细行程
```

#### 场景4: 供应商对账
```
1. 访问 /exchange-orders
2. 搜索特定供应商
3. 查看所有与该供应商的交换订单
4. 核对金额和付款状态
```
