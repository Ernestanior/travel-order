## 🎯 部署步骤

### 步骤 1: 在 Vercel 创建项目

1. 访问 [Vercel Dashboard](https://vercel.com/dashboard)
2. 点击 **"New Project"**
3. 选择 **"Import Git Repository"**
4. 找到并选择 `Ernestanior/travel-order`
5. 点击 **"Import"**

### 步骤 2: 配置项目

**Framework Preset**: Next.js（自动检测）

**Build Settings**:
- Build Command: `npm run build`
- Output Directory: `.next`
- Install Command: `npm install`

### 步骤 3: 配置环境变量

在 **"Environment Variables"** 部分添加：

```
DATABASE_URL=postgresql://neondb_owner:npg_mxyoVWE4hF7Y@ep-little-hat-aq2iyqb1-pooler.c-8.us-east-1.aws.neon.tech/neondb?channel_binding=require&sslmode=require
```

**或者使用 Vercel 的 Neon 集成**：
1. 在项目设置中，进入 **"Storage"** 标签
2. 点击 **"Connect Store"**
3. 选择 **"Postgres"** → **"Continue with Neon"**
4. 授权连接，环境变量将自动配置

### 步骤 4: 部署

1. 点击 **"Deploy"** 按钮
2. 等待构建完成（约 2-3 分钟）
3. 部署成功后会显示项目 URL

## 🚨 注意事项

### 1. 数据导入
- MDB 数据导入脚本**只能在本地运行**
- 需要安装 `mdbtools`（已通过 Homebrew 安装）
- 导入完成后数据会自动同步到 Neon 数据库
- Vercel 部署后会读取同一个 Neon 数据库

### 2. 生产环境配置
- 确保 Vercel 环境变量中的 `DATABASE_URL` 正确
- Prisma 会在构建时自动生成 client
- 数据库 schema 已同步到 Neon

### 3. 性能优化
- 使用 Neon 的连接池（URL 已包含 pooler）
- 考虑添加 Redis 缓存（未来优化）
- 图片优化使用 Next.js Image 组件
