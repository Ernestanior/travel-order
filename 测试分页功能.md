## 🔍 关键检查点

### Network 请求检查

打开浏览器开发者工具（F12）→ Network 标签

**正确的请求示例**:

```
# Booking Orders - 第 1 页
GET /api/booking-orders?searchType=all&page=1&limit=50
Response Size: ~50-100 KB
Status: 200

# Booking Orders - 第 2 页
GET /api/booking-orders?searchType=all&page=2&limit=50
Response Size: ~50-100 KB
Status: 200

# 按客户筛选
GET /api/booking-orders?searchType=customer&customer=ABC&page=1&limit=50
Response Size: < 100 KB
Status: 200
```

**❌ 错误的情况**:
- 同一个页面加载触发 2 次或更多相同请求
- 响应大小超过 1 MB（说明返回了所有数据）
- 请求没有 `page` 和 `limit` 参数

## 🐛 常见问题排查

### 问题 1: 页面加载很慢
**检查**:
- Network 标签中响应大小是否过大
- 是否没有分页参数
- 是否返回了所有数据

### 问题 2: 重复请求
**检查**:
- 是否触发了多次 API 请求
- useEffect 依赖项是否配置正确

### 问题 3: 分页不工作
**检查**:
- API 是否返回 `pagination` 对象
- 前端是否正确处理分页数据
- 页码变化时是否触发加载

### 问题 4: 筛选后显示错误
**检查**:
- 筛选后是否重置到第 1 页
- API 参数是否正确传递
